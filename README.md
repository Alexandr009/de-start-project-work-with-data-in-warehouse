
ETL-процессы:

актуализируются справочники (d_craftsman, d_product, d_customer);

обновляется таблица фактов (f_order);

инкрементально обновляется витрина данных customer_report_datamart с метриками по заказчикам за каждый месяц.

Основные шаги:

Объединение исходных данных – создаётся временная таблица tmp_sources с единым набором полей (заказы, мастера, товары, заказчики).

Обновление измерений – через MERGE обновляются/добавляются записи в d_craftsman, d_product, d_customer.

Подготовка фактов – создаётся tmp_sources_fact с внешними ключами на измерения, затем выполняется MERGE в f_order.

Инкрементальная витрина – на основе дельты (записи, изменившиеся после последней загрузки) пересчитываются агрегаты для customer_report_datamart.

Фиксация даты загрузки – в load_dates_customer_report_datamart сохраняется максимальная дата загрузки для следующего инкремента.
